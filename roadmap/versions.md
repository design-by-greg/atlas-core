# Versions

## Objectif

Definir la logique de versions du produit ATLAS.

## Regles officielles

Chaque version doit avoir un objectif clair, un perimetre controle et des criteres de validation.

Une version ne doit pas melanger trop de chantiers non stabilises.

## Fonctionnement

La logique de versionnement doit distinguer :

- fondations produit ;
- modules metier ;
- experience utilisateur ;
- automatisations ;
- IA ;
- QA ;
- documentation ;
- commercialisation SaaS.

Chaque version doit lister les modules inclus, les exclusions, les risques et les points a confirmer.

## Dependances

Les versions dependent de la roadmap, priorisation, QA, documentation, gouvernance et modules.

## Permissions

Toute version incluant une evolution des permissions doit avoir une validation QA specifique.

## Automatisations

Les automatisations doivent etre versionnees avec leurs conditions, limites et niveaux de validation.

## Points de vigilance

Ne pas livrer une version sans documentation.

Ne pas integrer une IA non encadree dans une version stable.

Ne pas publier une version si les tests critiques sont incomplets.

## A confirmer

- Noms officiels des versions.
- Contenu exact de la V1.
- Strategie de release publique.
