# Navigation

## Objectif

Definir les regles officielles de navigation dans ATLAS.

## Regles officielles

La navigation doit etre simple, stable et coherente entre les modules.

Un utilisateur doit toujours comprendre ou il se trouve, ce qu'il peut faire et comment revenir a l'information principale.

## Fonctionnement

La navigation doit organiser les modules par logique metier : tableau de bord, organisations, contacts, projets, devis, commandes, production, catalogue, facturation, portail client et parametres.

Les actions frequentes doivent etre accessibles rapidement.

## Dependances

La navigation depend des modules actifs, roles, permissions, workflows et configuration de l'organisation.

## Permissions

Un module non autorise ne doit pas apparaitre comme accessible.

Les actions visibles doivent respecter les permissions.

## Automatisations

La navigation peut proposer des raccourcis contextuels selon le statut, le role et les actions en attente.

## Points de vigilance

Ne pas multiplier les menus.

Ne pas cacher les actions essentielles.

Ne pas changer la logique de navigation d'un module a l'autre.

## A confirmer

- Structure finale du menu principal.
- Regles de navigation mobile.
