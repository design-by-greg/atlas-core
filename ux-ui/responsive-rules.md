# Responsive Rules

## Objectif

Definir les regles responsive officielles d'ATLAS.

## Regles officielles

ATLAS doit rester utilisable sur desktop, tablette et mobile lorsque le contexte s'y prete.

Les usages intensifs de gestion restent prioritaires sur desktop, mais les consultations et actions rapides doivent etre possibles sur mobile.

## Fonctionnement

Les interfaces doivent s'adapter : navigation, tableaux, formulaires, cartes, filtres, actions rapides et portail client.

Sur mobile, les tableaux peuvent devenir des listes de cartes simplifiees.

## Dependances

Les regles responsive dependent de la navigation, des tableaux, formulaires, dashboards, modules et portail client.

## Permissions

Les actions disponibles en mobile doivent respecter les memes permissions que sur desktop.

## Automatisations

Les raccourcis mobiles peuvent mettre en avant les actions urgentes ou frequentes.

## Points de vigilance

Ne pas tenter de reproduire exactement un ecran desktop complexe sur mobile.

Ne pas cacher une information critique sans alternative.

## A confirmer

- Priorite mobile par module.
- Breakpoints et comportements finaux.
