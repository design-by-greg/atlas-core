# API Tests

## Objectif

Definir les tests API officiels d'ATLAS.

## Regles officielles

Les API doivent etre testees pour verifier les entrees, sorties, erreurs, permissions et performances de base.

Une API ne doit pas exposer des donnees non autorisees.

## Fonctionnement

Les tests API doivent couvrir les routes principales, les donnees valides, les donnees invalides, les erreurs attendues, les droits d'acces et les cas limites.

Les contrats API doivent rester coherents avec la documentation.

## Dependances

Les tests API dependent des modules, permissions, validations, documentation developpeur et securite.

## Permissions

Chaque endpoint doit etre teste avec des profils autorises et non autorises.

## Automatisations

Les tests API doivent etre automatises autant que possible.

## Points de vigilance

Ne pas tester uniquement les reponses positives.

Ne pas oublier les erreurs de permission et de validation.
