# Functional Tests

## Objectif

Definir les controles fonctionnels d'ATLAS.

## Regles officielles

Chaque module doit etre verifie sur ses usages principaux et ses interactions avec les autres modules.

## Fonctionnement

Les controles portent sur creation, consultation, modification, validation, changement de statut, notification, erreur utilisateur et parcours complet.

## Dependances

Workflows, statuts, permissions, validations, modules et UX.

## Permissions

Les scenarios doivent etre verifies selon les roles concernes.

## Automatisations

Les controles repetitifs doivent etre automatises progressivement.

## Points de vigilance

Ne pas verifier uniquement le cas nominal.

## A confirmer

- Liste complete des scenarios V1.
- Parcours critiques prioritaires.
